# Лабораторна робота 7-9: Clicker Game

Ця лабораторна робота реалізує односторінковий інтерактивний застосунок (міні-гру) "Clicker Game", побудований на базі бібліотеки React та зібраний за допомогою Vite.

## Що робить програма

- Надає користувачеві можливість заробляти "кредити", клікаючи на головну кнопку.
- Дозволяє купувати покращення ("Курсор", "Автоклікер", "Майнер", "Ферма"), які збільшують або кількість кредитів за один клік, або пасивний дохід в секунду.
- Динамічно розраховує ціну кожного наступного покращення.
- ЩОСЕКУНДИ автоматично нараховує пасивний прибуток відповідно до куплених автоклікерів.
- Змінює стиль кнопок покращень (робить їх неактивними), якщо кредитів недостатньо для покупки.

## Основна логіка та React-складові (`App.jsx` та компоненти)

- **Стан компонента (`useState`)**
  - Використовується для зберігання поточного балансу кредитів (`credits`).
  - Використовується для збереження доходу за клік (`clickPower`) та пасивного доходу (`autoIncome`).
  
- **Ефекти життєвого циклу (`useEffect`)**
  - Реалізує таймер через `setInterval`, який кожну секунду додає `autoIncome` до загальної суми `credits`.
  - Таймер правильно очищується через функцію очищення (`clearInterval`) для запобігання витоку пам'яті.

- **Обробка подій**
  - Клік по головній кнопці викликає функцію, що збільшує `credits` на розмір `clickPower`.
  - Клік по кнопці покращення викликає функцію покупки, яка віднімає відповідну суму з балансу та оновлює `clickPower` або `autoIncome`.

## Структура React-застосунку

- `<div className="stats">`: Контейнер для відображення поточної статистики (кредити, сила кліку, автоматичний дохід).
- `<button className="main-btn">`: Головна кнопка "Клік!", з якою взаємодіє гравець.
- `<div className="upgrades">`: Контейнер із кнопками для покупок різноманітних поліпшень. Усі кнопки мають атрибут `disabled={credits < cost}`, що унеможливлює купівлю без достатньої суми.

## Додатково

Додаток має темну тему та адаптований дизайн, що прописаний в `index.css` та `App.css`. Він наочно демонструє базові хуки React та реактивність інтерфейсу без прямого маніпулювання DOM.
