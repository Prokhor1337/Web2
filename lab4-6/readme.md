# Лабораторна робота 4-6: Фільмова база

## 1. Мета роботи та опис завдання
Метою цієї лабораторної роботи є створення односторінкового застосунку «Фільмова база», який працює з віддаленим сервером через API, дозволяє фільтрувати, сортувати дані та розбивати їх на сторінки.

Робота повністю відповідає наступним вимогам:
- Асинхронне отримання даних з TV Maze API за допомогою `fetch` та `async/await`.
- Динамічне відображення даних у вигляді карток за допомогою DOM-методів.
- Використання сучасного стандарту ES6 (деструктуризація, стрілочні функції, шаблонні рядки).
- Реалізація фільтрації (за назвою) та сортування (за алфавітом та рейтингом).
- Належна обробка помилок `try/catch`.
- **Додатково**: реалізовано пагінацію (посторінковий вивід) та лічильник знайдених результатів.

---

## 2. Отримання даних з API та обробка помилок
Завантаження даних реалізовано в асинхронній функції `fetchMovies()`:
1. Використовується блок `try/catch` для перехоплення мережевих помилок.
2. Робиться запит до `https://api.tvmaze.com/shows` за допомогою `await fetch(API_URL)`.
3. Перевіряється статус відповіді: `if (!response.ok)`. Якщо сервер повернув помилку (наприклад, 404), генерується виключення `throw new Error(...)`.
4. Якщо помилок немає, дані перетворюються у JSON формат (`await response.json()`) та зберігаються у глобальний масив `allMovies`.
5. У разі перехоплення помилки блоком `catch`, викликається функція `showError(message)`, яка змінює клас CSS-контейнера `#errorMessage` і виводить текст проблеми безпосередньо на екран користувача.

## 3. Відображення даних, Пагінація та Лічильник
Функція `displayMovies(movies)` відповідає за рендеринг:
- Спочатку вона очищає попередній вміст `moviesContainer.innerHTML = ""`.
- **Лічильник**: Оновлює текстовий вміст елемента `#movieCounter`, показуючи кількість фільмів у переданому масиві (`movies.length`).
- **Пагінація (Посторінковий вивід)**:
  - Визначається `startIndex` та `endIndex` на основі поточної сторінки (`currentPage`) та кількості елементів на сторінку (`itemsPerPage = 12`).
  - Метод `movies.slice(startIndex, endIndex)` відрізає лише ті фільми, які мають бути показані зараз.
- За допомогою `forEach` відбувається обхід відфільтрованого масиву. Для розпакування об'єкта фільму застосовано **ES6 деструктуризацію**: `const { name, image, rating, genres } = movie`.
- Формується HTML-рядок картки за допомогою шаблонних рядків (Template literals). Якщо у фільму відсутній постер, використовується оператор `? :` для підстановки заглушки (`placeholder`).
- Картки вставляються у DOM методом `insertAdjacentHTML`.
- Після відмальовування карток викликається `renderPagination()`, яка динамічно створює кнопки "Попередня" та "Наступна". Кнопки блокуються (`disabled = true`), якщо користувач знаходиться на першій або останній сторінці.

## 4. Реалізація функціоналу фільтрації та сортування
Логіка обробки даних винесена у функцію `handleProcessData(resetPage)`:
- При будь-якій зміні у полі пошуку або зміні типу сортування, поточна сторінка пагінації скидається на першу (`currentPage = 1`).
- **Фільтрація**: Використовується метод масиву `filter()`. Він порівнює назву кожного фільму (`movie.name.toLowerCase()`) з введеним пошуковим запитом (`searchInput.value.toLowerCase()`) за допомогою методу `includes()`.
- **Сортування**: Використовується метод `sort()`.
  - За алфавітом: порівняння відбувається за допомогою нативного методу `localeCompare()`, що гарантує правильне сортування текстових значень.
  - За рейтингом: фільми сортуються від більшого рейтингу до меншого (спадання). Оскільки не всі шоу мають рейтинг, використовується `|| 0`, щоб уникнути помилок `NaN`.

## 5. Базова сторінка та UI
HTML сторінка містить усі необхідні контейнери:
- Панель керування `.controls` з текстовим `<input>` та випадаючим списком `<select>`.
- Контейнер для повідомлень про помилки `#errorMessage`.
- Лічильник знайдених фільмів `<p id="movieCounter">`.
- Грід-контейнер для карток `#moviesContainer`.
- Блок для кнопок навігації по сторінках `<div id="pagination">`.

У файлі `laboratory4-6.css` налаштовано адаптивну сітку (`display: grid`) для красивого відображення карток та стилізовано кнопки пагінації.
