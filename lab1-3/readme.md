# Лабораторна робота 1-3: Конструктор резюме

Ця лабораторна робота реалізує односторінковий застосунок «Конструктор резюме», який використовує принципи ООП (класи, наслідування) для створення структурованого резюме з введених користувачем даних.

## Що робить програма

- Запитує у користувача дані (Особиста інформація, Досвід роботи, Освіта, Навички) через `prompt`.
- Перевіряє та валідує введені дані перед створенням резюме.
- Створює об'єкти відповідних класів для кожної секції.
- Об'єднує секції за допомогою головного класу `Resume`.
- Динамічно рендерить зібране резюме на HTML-сторінку.

## Основні функції та класи файлу `lab1.js`

- `dataProcessor(transformer)(validator)(errorMessage)(value)`
  - Каррована функція-фабрика для перевірки даних (інкапсулює валідацію).
  - Створює специфічні валідатори, такі як `parseAge`, `validateStr`, `validateYear`.

- `ResumeSection` (Базовий клас)
  - Клас, який містить заголовок секції (`title`) та базовий метод `render()` для генерації HTML-розмітки.
  - Використовує геттери та сеттери з валідацією.

- `PersonalInfo`, `Experience`, `Education`, `Skills` (Дочірні класи)
  - Наслідуються від `ResumeSection`.
  - Мають власні властивості (наприклад, вік, посада, університет) та перевизначають метод `render()` за допомогою `super.render()` для додавання специфічних даних.

- `Resume` (Головний клас)
  - Має метод `addSection()` для агрегації створених секцій.
  - Відповідає за рендеринг усього резюме в обраний DOM-контейнер (`resumeOutput`).

## Структура HTML (`index.html`)

- Кнопка `#createResumeBtn` для запуску збору даних.
- Контейнер `#resumeOutput` для відображення згенерованого резюме.

## Додатково

Резюме стилізоване за допомогою файлу `style.css`.
При генерації також використовується `localStorage` для збереження готового резюме (серіалізується у формат JSON).
