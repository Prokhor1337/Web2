# Лабораторна робота 1-3: Конструктор резюме

## 1. Мета роботи та опис завдання
Метою цієї роботи є створення односторінкового інтерактивного застосунку «Конструктор резюме». Застосунок дозволяє користувачеві вводити свої особисті дані, досвід роботи, освіту та навички, після чого генерує та відображає готове резюме на веб-сторінці.

Робота повністю відповідає наступним вимогам:
- Використання основ JavaScript (strict mode).
- Застосування замикань та каррінгу для перевірки даних.
- Використання об'єктно-орієнтованого програмування (класи, наслідування, інкапсуляція).
- Робота з DOM та інтерактивність через `prompt` і `alert`.
- Збереження результатів у `localStorage`.

---

## 2. Планування та структура (Базова сторінка)
Для реалізації проекту використовується базова HTML-сторінка (`index.html`), яка містить:
- Кнопку `<button id="createResumeBtn">` для ініціалізації процесу збору даних.
- Порожній контейнер `<div id="resumeOutput"></div>` куди динамічно генерується результат.
- Підключений CSS файл `style.css` для візуального оформлення карток секцій резюме.
- Підключений JavaScript файл `lab1.js` з атрибутом `defer` та використанням `"use strict"`.

## 3. Модуль введення та валідації даних (Функції, замикання, каррінг)
Збір даних реалізовано за допомогою нативних браузерних вікон `prompt`.
Для того, щоб забезпечити надійну валідацію введених даних, розроблено універсальний **каррований фабричний метод**:

```javascript
const dataProcessor = (transformer) => (validator) => (errorMessage) => (value) => {
    const transformedValue = transformer(value);
    if (validator(transformedValue)) return transformedValue;
    throw new Error(errorMessage);
};
```
Цей підхід (каррінг) дозволяє створювати вузькоспеціалізовані функції для перевірки конкретних типів даних. Наприклад:
- `parseAge`: перетворює ввід на `Number` і перевіряє, щоб вік був від 1 до 120 років.
- `validateStr`: перевіряє, щоб текстове поле не було порожнім після видалення пробілів (`trim()`).
- `validateYear`: перевіряє коректність введених років досвіду (не від'ємне число).

Якщо валідація не проходить, генерується виключення (`throw new Error`), яке згодом перехоплюється блоком `try/catch`, інформуючи користувача через `alert`.

## 4. OOP-модуль для створення резюме (Класи, наслідування)
Архітектура застосунку побудована на принципах ООП (Об'єктно-орієнтованого програмування). 

### Базовий клас `ResumeSection`
Усі розділи резюме наслідуються від єдиного базового класу.
- **Інкапсуляція**: містить захищену властивість `_title`.
- **Геттери/Сеттери**: використовує `get title()` та `set title()` з вбудованою валідацією.
- **Метод `render()`**: створює базовий HTML-елемент `<div>` із заголовком секції.

### Дочірні класи (`PersonalInfo`, `Experience`, `Education`, `Skills`)
Вони наслідують (`extends`) базовий клас `ResumeSection`.
- Викликають батьківський конструктор через `super(title)`.
- Мають власні специфічні поля (наприклад, `jobTitle`, `company`, `degree`).
- **Поліморфізм**: перевизначають метод `render()`. Вони викликають `super.render()`, отримують базовий HTML-елемент, а потім додають туди власні специфічні дані, форматуючи їх у вигляді параграфів `<p>`.

### Головний клас `Resume`
- Містить масив `this.sections` для агрегації всіх блоків резюме.
- Має метод `addSection(section)`, який перевіряє за допомогою `instanceof`, чи є переданий об'єкт нащадком `ResumeSection`.
- Метод `render(containerId)` проходить циклом `forEach` по всім зібраним секціям, викликає їхній власний `render()` і вставляє готовий HTML у DOM.

## 5. Відображення та збереження результатів (Інтеграція)
Після натискання на кнопку "Створити резюме" спрацьовує обробник подій:
1. Запускається блок `try/catch`.
2. Користувачу по черзі виводяться `prompt` для збору інформації (ім'я, пошта, вік, освіта, навички).
3. Створюються об'єкти відповідних класів, і відбувається автоматична валідація даних через сеттери.
4. Якщо користувач ввів некоректні дані (наприклад, порожній рядок замість імені), викидається помилка, і процес створення переривається з виводом `alert("Помилка: ...")`.
5. Якщо все правильно, викликається метод `myResume.render("resumeOutput")`.
6. **Додаткове завдання (localStorage)**: під час рендеру всі зібрані секції серіалізуються у формат JSON і зберігаються у `localStorage.setItem("savedResume", ...)`, що дозволяє не втратити дані.
